More Dell XPS m1210 Problems

Posted on August 24, 2007 by jim Posted in Uncategorized — No Comments ↓

A while ago I blogged about the poor networking support on my XPS m1210 laptop. Since then I’ve installed Vista on that machine and have watched the performance go from reasonable to abysmal with no real gain in functionality.

Two days ago I became really frustrated with watching the hard disk light and ordered 4GB of memory to try to satisfy Vista’s hunger. Although I typically only run Outlook, Visual Studio 200x, and SQL Server on this laptop it has felt like wading through treacle every time I try to do anything with it. 

So yesterday I unwillingly upgraded the memory on my XPS m1210 from the existing 2GB to 4 GB and duly leveled my box and installed Vista 64 bit to take advantage of it. I was a bit narked to see that even with the 64 bit OS that Vista can only use around 3.3GB of the 4GB of memory installed.

Looking into the Dell’s BIOS revealed this isn’t a Vista problem per se, but the BIOS itself is not releasing all of the RAM. The “available memory” it’s reporting is about the 3.3GB that Vista is seeing. Sensing it was time for a BIOS upgrade to the latest version (A08) I duly downloaded and flashed my BIOS. Still no joy, the BIOS is still only allowing me 3.3GB or so to use, blatantly and mockingly so since it acknowledges there is a full 4GB physically in place.

So it seems that no matter what I do I can’t get access to all the memory I’ve bought. I really would have been hopping mad if I’d bought it with 4GB from Dell last year when memory was really expensive!

I am so unhappy about this situation. Years ago when I bought my first Dell I vowed I would never do it again (poor build quality and worse customer service were the culprits then), and yet I succumbed. Next year when it’s time to upgrade this laptop I won’t be going with Dell. In fact given I’ve been  underwhelmed by Vista I don’t know if I will be sticking with Microsoft either especially now the .Net dev tools I use running happily in under virtualisation on Ubuntu and Mac.

On the other hand if anyone has any ideas about how to rectify the situation (which doesn’t include Google or reminding me to upgrade to a 64 bit OS) then drop me a line…)
